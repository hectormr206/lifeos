# Axi Visual System and Wallpaper Strategy

Documento maestro para definir el papel del **ajolote** dentro de la identidad de LifeOS y traducirlo a wallpapers, iconos, landing page y superficies publicas.

**Version:** 1.0.0
**Updated:** 2026-04-01
**Status:** Official Direction

---

## 1. Decision Central

LifeOS debe tener una identidad visual reconocible que no dependa solo de colores o tipografia.

La decision de marca es esta:

- **LifeOS** es la marca principal del sistema operativo
- **Axi** es la cara viva del proyecto y del asistente
- **el ajolote mexicano** es el motivo simbolico central del universo visual

Esto significa que LifeOS no debe buscar un animal distinto por release como mecanica principal.
La estrategia correcta es:

- **un simbolo central consistente**
- **muchas interpretaciones coherentes**

Eso vuelve a LifeOS mas reconocible a largo plazo.

---

## 2. Tesis de Marca

El ajolote no debe usarse como adorno arbitrario.
Debe funcionar como una extension natural de lo que LifeOS quiere representar:

- identidad mexicana real
- origen claro sin exclusividad geografica
- tecnologia cercana y no intimidante
- inteligencia local y resiliente
- rareza valiosa y dificil de reemplazar
- una criatura que vive, observa, recuerda y se adapta

En otras palabras:

- Apple tiene la manzana
- Android tiene el robot
- OpenClaw tiene el cangrejo
- **LifeOS debe tener al ajolote**

---

## 3. Arquitectura de Marca

### 3.1 LifeOS

La marca del sistema:

- nombre del proyecto
- wordmark
- logo del sistema
- identidad general del OS

### 3.2 Axi

La interfaz viva de la marca:

- asistente
- Telegram avatar
- tray icon
- onboarding
- overlays
- ilustraciones hero

### 3.3 El Ajolote

El fundamento narrativo y visual:

- origen mexicano
- referencia biologica y cultural
- figura madre para wallpapers, artes y piezas publicas

Esto evita un error comun:

- que Axi parezca una mascota separada de LifeOS

No debe ser asi.
Axi es la manifestacion mas personal del mismo sistema visual.

---

## 4. Principios de Uso

### HACER

- usar al ajolote como motivo central de identidad
- mantener a Axi reconocible entre superficies
- construir una estetica elegante, tecnica y calmada
- conservar el tono dark-first de LifeOS
- usar referencias mexicanas con respeto y fundamento
- dejar claro que LifeOS esta hecho en Mexico y abierto a una comunidad global
- preferir simbolos, texturas, patrones y siluetas antes que disfraces literales

### NO HACER

- convertir a Axi en caricatura infantil
- usar trajes regionales genericos sin investigacion
- volver cada wallpaper una postal turistica
- romper las proporciones o personalidad de Axi
- rediseñar a Axi desde cero en cada pieza
- usar referencias culturales superficiales o estereotipadas
- presentar la identidad mexicana como limite de a quien pertenece el proyecto

---

## 5. Surfaces Donde Debe Vivir Axi

El ajolote no debe limitarse a una sola ilustracion.
Debe aparecer con distinta intensidad segun la superficie.

### Presencia Alta

- wallpaper default
- landing page hero illustration
- Telegram icon/avatar
- onboarding
- splash visuals

### Presencia Media

- lock screen
- greeter
- about screen
- release art
- social cards

### Presencia Baja

- tray icon
- badges
- empty states
- stickers o micro-ilustraciones

## 6. Decision para Wallpapers

### 6.1 Regla General

El wallpaper default de LifeOS debe incluir un ajolote reconocible.

No como clipart.
No como dibujo infantil.
No como personaje aislado sin atmosfera.

Debe sentirse como:

- LifeOS
- Axi
- Mexico
- tecnologia soberana

### 6.2 Jerarquia de Wallpapers

LifeOS debe tener esta jerarquia:

1. **Wallpaper canonico**
2. **Variantes de sistema**
3. **Coleccion regional**
4. **Ediciones especiales**

### 6.3 Wallpaper Canonico

El wallpaper canonico debe ser:

- un ajolote claramente identificable
- dark-first
- elegante
- atmosferico
- usable diario
- sin texto
- sin exceso de detalle ruidoso

Este es el wallpaper que la gente deberia asociar inmediatamente con LifeOS.

### 6.4 Variantes de Sistema

Derivadas directas del canonico:

- `default`
- `minimal`
- `lock-screen`
- `greeter`
- `light`

Todas deben pertenecer a la misma familia visual.

## 7. Estrategia Regional

### Decision

La coleccion regional debe empezar por **estados**, no por ciudades.

### Por que estados primero

- son menos y mas manejables
- suelen tener simbolos culturales mas documentados
- es mas facil sostener consistencia visual
- evita una explosion de piezas imposibles de investigar bien

### Ciudades despues

Las ciudades deben reservarse para:

- ediciones especiales
- founder editions
- colaboraciones
- aniversarios

Esto deja espacio para una pieza personal sin forzar una taxonomia imposible.

---

## 8. Modelo Visual para Regionalizar Axi

No se debe resolver todo con "ponerle ropa tradicional".

La formula recomendada para cada edicion regional es:

1. **un motivo cultural**
2. **un motivo geografico o arquitectonico**
3. **una micro-paleta derivada**
4. **una atmosfera**
5. **una presencia clara del ajolote**

### Ejemplos validos de motivo

- bordado
- cantera
- agua
- desierto
- niebla
- flora local
- geometria artesanal
- arquitectura emblematicamente local

### Ejemplos a evitar

- sombreros puestos por default
- disfraces sin contexto
- simbolos patrios metidos a fuerza
- collage turistico
- saturacion folklorica

---

## 9. Coleccion Inicial Recomendada

### 9.1 Axi Xochimilco

**Rol:** wallpaper canonico de LifeOS

**Por que primero:**

- el ajolote esta cultural y biologicamente ligado a Xochimilco
- da una raiz mexicana clara y defendible
- conecta con agua, reflejo, noche y vida local

**Motivos visuales sugeridos:**

- agua oscura con reflejos teal
- vegetacion acuática sutil
- niebla ligera
- bioluminiscencia suave
- ajolote central o semi-emergente

### 9.2 Axi Querétaro

**Rol:** primera edicion estatal

**Por que:**

- representa bien tu raiz personal
- Querétaro tiene lenguaje visual artesanal fuerte
- hay una via elegante mediante referencias otomies y de Amealco

**Motivos visuales sugeridos:**

- patrones textiles sobrios
- geometria inspirada en bordado
- rosa y teal con neutros oscuros
- cantera o silueta sutil del semidesierto

### 9.3 Axi San Juan del Río Founder Edition

**Rol:** edicion especial personal

**Por que:**

- conecta directamente contigo como fundador
- puede funcionar como pieza narrativa y no solo decorativa
- debe sentirse mas intima y menos "coleccion oficial base"

**Motivos visuales sugeridos:**

- referencia sutil al Puente de la Historia
- textura mineral o cantera
- atmosfera de provincia historica
- composicion mas contemplativa

## 10. Telegram y Axi

La recomendacion oficial es:

- el icono principal de Telegram debe ser **Axi**

No el logo textual de LifeOS.
No un icono generico.

### Requisitos del avatar de Telegram

- close-up legible en tamaño pequeño
- ojos claros y expresivos
- branquias distinguibles
- silueta limpia
- contraste alto
- fondo simple o degradado muy controlado

### Lo que debe comunicar

- esta cuenta pertenece a LifeOS
- hay un asistente real del sistema
- el proyecto tiene un rostro propio

---

## 11. Landing Page y Presencia Publica

La landing page debe usar el branding existente de LifeOS y reconocer de forma sutil el origen del ajolote.

Eso significa:

- no volver la home una pagina "folklorica"
- no ocultar que Axi es un ajolote mexicano
- no inventar una mascota distinta para web

La web debe sentirse como:

- una extension publica de LifeOS
- con el ajolote como eje simbolico
- con sobriedad tecnica y cultural

---

## 12. Reglas para Produccion de Wallpapers

Cada wallpaper oficial debe registrar:

- nombre
- serie
- region
- motivacion
- referencias usadas
- paleta
- superficie destino

### Naming recomendado

- `lifeos-axi-xochimilco-default`
- `lifeos-axi-xochimilco-lock`
- `lifeos-axi-queretaro-edition`
- `lifeos-axi-sanjuan-founder`

### Resoluciones minimas

- `3840x2160` para 4K
- preparar versiones de recorte para `16:10` y `ultrawide`

### Salidas recomendadas

- wallpaper hero
- version minimal
- lock
- greeter
- social preview opcional

---

## 13. Hoja de Ruta Recomendada

### Etapa 1

- fijar a Axi como icono oficial de Telegram
- definir wallpaper canonico `Axi Xochimilco`
- actualizar landing para reflejar el sistema visual

### Etapa 2

- producir familia base de wallpapers de sistema
- producir `Axi Querétaro`
- documentar pipeline visual

### Etapa 3

- abrir serie estatal
- lanzar ediciones especiales como `San Juan del Río Founder Edition`
- explorar colecciones limitadas por release o colaboracion

---

## 14. Criterio de Calidad

Una pieza visual de LifeOS solo esta aprobada si:

- sigue pareciendo LifeOS
- sigue pareciendo Axi
- sigue pareciendo un ajolote
- no cae en infantilizacion
- no cae en turismo decorativo
- puede vivir junto al desktop sin cansar
- tiene suficiente identidad para ser recordada
